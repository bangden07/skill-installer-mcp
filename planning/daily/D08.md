## Decisions

Keputusan penting hari ini:
- MCP handler yang belum punya core implementation tetap diexpose sebagai `NOT_IMPLEMENTED` placeholder agar permukaan tool sudah jelas
- `register-tools` sekarang menjadi tempat wiring dependency default untuk local-only runtime awal
- local source dijadikan baseline runtime pertama agar flow bisa diuji tanpa network dependency

## Problems

Masalah yang muncul:
- MCP server masih placeholder dan belum memakai SDK MCP sungguhan
- handler `list_installed_skills` masih mengandalkan data manifest tanpa enrichment tambahan
- beberapa feature seperti `analyze/recommend/update` masih placeholder
- ada beberapa compile error kecil setelah bootstrap runtime pertama dijalankan
- smoke flow pertama mengungkap asumsi bahwa `planInstall` menuntut canonical dir sudah ada
- validasi fingerprint plan juga sempat tidak konsisten antara plan dan install
- Node test dengan `--experimental-strip-types` tidak cocok dengan import `.js` ESM project ini
- flow `update` perlu tetap konservatif karena source remote belum ada; implementasi sekarang fokus ke tracked local source dan reapply target yang sudah ada

## Mitigation

Cara mengatasi atau langkah lanjut:
- lanjutkan ke bootstrap runtime `package.json` dan `tsconfig.json`
- mulai smoke test local install path
- setelah itu tambahkan git source dan test dasar
- compile error kecil diperbaiki lewat penyesuaian type `modeResolved`, signature placeholder handler, dan union type pada hashing util
- `resolveTarget` di base adapter dilonggarkan agar planning tidak gagal sebelum canonical install terjadi
- penyimpanan `requestedSkills` pada plan diselaraskan dengan normalisasi selector yang dipakai saat install validation
- test dijalankan dari hasil build `dist/` agar tetap konsisten dengan strategi import `.js` pada source TypeScript ESM
- `update_skills` tahap sekarang hanya memperbarui skill yang sudah tracked di manifest dan memakai source resolver/fetcher yang sudah tersedia tanpa menambah scope registry baru
